============================================================
  SesliKitap v2 — Optimizasyon Planı
============================================================

DURUM: Önceki turda P0 buglar (çevrimdışı okuma, kırık importlar,
memory leak, debounce) ve P1/P2 iyileştirmeler (TTS feedback, yardım
komutu, font/touch, kod temizliği) tamamlandı. Aşağıdaki yeni
sorunlar kapsamlı kod analizi ile tespit edildi.

============================================================
  FAZ 1 — P0: KRİTİK BUGLAR
============================================================

[ ] P0-1: Sayfa bölme tutarsızlığını gider
    Dosya: src/screens/ReaderScreen.tsx
    Sorun: ReaderScreen'de naif 2000-char sabit bölme var.
           useOfflineBooks.ts'de akıllı paragraf/cümle/kelime bölme var.
           Online kitaplar aptal bölücüyü, offline kitaplar akıllısını kullanıyor.
    Çözüm: ReaderScreen'deki lokal splitIntoPages ve CHARS_PER_PAGE
           sabitini sil, useOfflineBooks'tan import et.

[ ] P0-2: Swipe ile sayfa değişiminde lastPage kaydedilmiyor
    Dosya: src/screens/ReaderScreen.tsx
    Sorun: swipeHandler setCurrentPage çağırıyor ama updateLastPage
           çağırmıyor. Uygulama yeniden başlatılınca konum kaybolur.
    Çözüm: Swipe handler'a updateLastPage(bookId, newPage) çağrısı ekle.

[ ] P0-3: TTS auto-advance'de lastPage kaydedilmiyor
    Dosya: src/screens/ReaderScreen.tsx
    Sorun: readFromWord otomatik sayfa geçişinde updateLastPage yok.
    Çözüm: Auto-advance sonrası updateLastPage(bookId, currentPage + 1)
           ekle. bookId'yi dependency array'e ekle.

[ ] P0-4: Gemini JSON parse güvenliği
    Dosya: src/utils/geminiService.ts
    Sorun: JSON.parse(jsonStr) — genel try-catch'te yakalanır ama
           conversation history tutarsız kalır ve kullanıcı yanıt alamaz.
    Çözüm: İç try-catch ile sarma. Parse başarısız olursa regex ile
           speech alanını çıkarmayı dene. Bulunamazsa ham metni oku.

[ ] P0-5: Voice loading race condition
    Dosya: src/utils/tts.ts
    Sorun: loadBestVoice() modül yüklenirken await'siz çağrılıyor.
           İki eşzamanlı çağrı voicesLoaded kontrolünü geçip ikisi de
           voice listesi çekebilir.
    Çözüm: voicesLoaded boolean yerine loadPromise: Promise<void> | null
           kullan. İlk çağrı promise oluşturur, sonrakiler bekler.

[ ] P0-6: Sayfa cache'i sınırsız büyüyor
    Dosya: src/hooks/useOfflineBooks.ts
    Sorun: pageCache Map'i asla temizlenmiyor (deleteBook hariç).
           100 kitap = devasa bellek kullanımı.
    Çözüm: MAX_CACHED_BOOKS = 3 sabiti ekle. Yeni giriş eklerken
           en eski girişi sil (LRU).

============================================================
  FAZ 2 — P1: HATA YÖNETİMİ VE UX
============================================================

[ ] P1-1: Tüm API çağrılarına timeout ekle
    Yeni dosya: src/utils/fetchWithTimeout.ts
    Değişecek: gutenbergAPI.ts, wikisourceAPI.ts, geminiService.ts
    Çözüm: AbortController ile fetchWithTimeout(url, options, timeoutMs)
           fonksiyonu yaz. Gutenberg/Wikisource: 10s, Gemini: 15s.
           Timeout hatası kullanıcıya sesli bildirilir.

[ ] P1-2: TTS hatalarını yutmayı bırak
    Dosya: src/utils/tts.ts
    Sorun: onError: () => resolve() — TTS çökerse kullanıcı habersiz.
    Çözüm: Hata logla. speak() için reject et. speakBook() için
           konsola yaz ama resolve et (okuma akışını bozmamak için).

[ ] P1-3: bookStorage JSON.parse koruması
    Dosya: src/store/bookStorage.ts
    Sorun: AsyncStorage verisi bozuksa JSON.parse çöker.
    Çözüm: getBooks()'ta try-catch. Parse hatasında boş dizi dön,
           bozuk veriyi temizle.

[ ] P1-4: Kitap silmeden önce onay iste
    Dosya: src/screens/LibraryScreen.tsx
    Sorun: Tek dokunuşla kalıcı silme — kör kullanıcı için tehlikeli.
    Çözüm: Alert.alert() ile onay diyaloğu. Diyalog öncesi TTS ile
           "silinecek, emin misiniz?" sor.

[ ] P1-5: Wikisource markup temizliğini iyileştir
    Dosya: src/utils/wikisourceAPI.ts
    Sorun: İç içe {{}} ve [[]] düzgün temizlenmiyor.
    Çözüm: İteratif while döngüsü ile en içtekinden dışa doğru temizle.
           <ref>, [[Kategori:]], __TOC__ gibi öğeleri de temizle.

[ ] P1-6: Okuma ilerleme bildirimi
    Dosyalar: tts.ts, ReaderScreen.tsx, geminiService.ts
    Çözüm: announce.progress(current, total) fonksiyonu ekle.
           Gemini'ye "neredeyim" -> progress aksiyonu ekle.
           Sayfa bilgi alanına dokunulduğunda da tetikle.

[ ] P1-7: Uyku zamanlayıcısı
    Dosyalar: ReaderScreen.tsx, tts.ts, geminiService.ts
    Çözüm: "uyku zamanlayıcı X dakika" komutu -> set_timer aksiyonu.
           setTimeout ile pauseReading tetikle. Unmount'ta timer temizle.

[ ] P1-8: Okuma hızı kalıcılığı
    Dosyalar: tts.ts, bookStorage.ts
    Çözüm: AsyncStorage'da @seslikitap_speed key'i ile hızı kaydet.
           Uygulama başlangıcında oku.

[ ] P1-9: Yer imi desteği
    Dosyalar: bookStorage.ts, ReaderScreen.tsx, geminiService.ts, tts.ts
    Çözüm: BookMetadata'ya bookmarks?: number[] ekle.
           "yer imi ekle" / "yer imlerim" / "yer imine git" komutları.
